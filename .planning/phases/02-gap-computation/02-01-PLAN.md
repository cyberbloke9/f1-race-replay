**Algorithm design:**
Each driver's resampled data has `lap` (lap number) and `rel_dist` (0-1 around track) on a common timeline.
Define `progress = (lap - 1) + rel_dist` — a continuous float representing laps completed.
For gap computation at frame `i`:
1. Compute progress for all drivers
2. Sort by progress descending → positions
3. For each driver behind the leader: find the most recent frame where the leader had the same progress value. Gap = timeline[i] - timeline[j]. Use `np.searchsorted` on the leader's progress array.
4. For interval gaps: same logic but with the car directly ahead.

Edge cases:
- Leader gap is always 0.0
- Lapped cars (progress > 1 lap behind): show "+1 LAP" instead of seconds
- First few frames (no history): gap = None
- Cars on same progress (side by side): gap ≈ 0.0

  <behavior>
    compute_driver_gaps(resampled_data: dict, timeline: np.ndarray) -> dict
    - Input: resampled_data is {driver_code: {"lap": array, "rel_dist": array, ...}}, timeline is the common time array
    - Output: {driver_code: {"gap_to_leader": array, "gap_to_ahead": array}} — arrays same length as timeline
    - Leader's gap_to_leader is all zeros
    - Gap values are in seconds (float), None/NaN for unavailable
    - Lapped cars: gap value is negative (convention: -1.0 per lap behind)

    Cases:
    - 2 drivers, one 0.5s ahead → gap_to_leader ≈ 0.5 for behind driver
    - 3 drivers in order → gap_to_ahead[P2] = gap to P1, gap_to_ahead[P3] = gap to P2
    - Driver lapped (>1 full lap behind) → gap_to_leader uses lap count
    - Single driver → gap_to_leader = 0.0, gap_to_ahead = 0.0
    - Early frames with no history → gap = NaN
  </behavior>
  <implementation>
    1. For each frame, compute progress = (lap - 1) + rel_dist for all drivers
    2. Sort drivers by progress descending → leader is first
    3. For leader's gap: build leader's progress timeline as sorted array
    4. For each non-leader driver at frame i: use np.searchsorted on leader's progress history to find when leader was at same progress. Gap = timeline[i] - timeline[j].
    5. For interval: same but compare to car directly ahead in position order.
    6. Vectorize where possible — avoid Python loops over frames (use numpy operations).

    Helper function: _find_time_at_progress(progress_timeline, time_timeline, target_progress) -> float
    Uses searchsorted + linear interpolation between adjacent frames.
  </implementation>
</feature>
