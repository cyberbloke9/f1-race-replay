    ```python

    def stream_server():
        server = TelemetryStreamServer(host='localhost', port=19876)  # High port, avoid conflicts

        time.sleep(0.1)  # Let server thread start

    ```

    **Tests:**

    1. **test_server_starts_and_accepts_connections** — Start server fixture. Connect a plain `socket.socket()` client. Verify connection succeeds (no exception). Close client.

    2. **test_server_broadcasts_json_to_client** — Start server, connect plain socket client. Call `server.broadcast(sample_telemetry_frame)`. Read from client socket. Verify received data is valid JSON matching the broadcast dict. Remember: messages end with `\n`.

    3. **test_server_broadcasts_to_multiple_clients** — Connect 2 plain socket clients. Broadcast once. Verify both clients receive the same data.

    4. **test_server_handles_disconnected_client** — Connect 2 clients, close one. Broadcast — should succeed without error. Remaining client should receive data. Dead client should be removed from server.clients list.

    5. **test_server_stop_closes_all** — Start server, connect client. Call `server.stop()`. Verify `server.running` is False. Verify `server.clients` is empty.

    **Important:** Use `time.sleep(0.2)` after connect/broadcast to let threading catch up. Use `socket.settimeout(2.0)` on test clients to avoid hanging. Wrap socket operations in try/finally to ensure cleanup even on test failure.

<task type="auto">
  <name>Task 2: Test TelemetryStreamClient parsing logic</name>
  <files>tests/test_stream.py (append)</files>
  <action>
    Add client tests to the same file. Since TelemetryStreamClient inherits QThread and uses PySide6 Signals, we test the parsing logic without running the full Qt event loop.

    **Approach:** Mock the QThread base and Signal emissions. Test `_receive_data()` by mocking `self.socket.recv()` to return prepared byte chunks.

    **Tests:**

    1. **test_client_parses_complete_json_message** — Create a TelemetryStreamClient instance, but mock its signals (data_received, connection_status, error_occurred) with MagicMock. Mock `self.socket` with a MagicMock. Set `self.connected = True`, `self.running = True`. Mock `socket.recv()` to return `json.dumps(sample_frame).encode() + b'\n'` on first call, then raise `Exception("stop")` on second call. Call `client._receive_data()` (it will loop once then break on exception). Verify `data_received.emit` was called with the parsed dict.

    2. **test_client_handles_chunked_data** — Mock recv to return the JSON message split across 2 chunks (first half, then second half + newline). Verify data_received.emit is called once with the complete parsed dict after both chunks arrive.

    3. **test_client_handles_multiple_messages** — Mock recv to return two JSON messages in one chunk (msg1\nmsg2\n). Verify data_received.emit is called twice.

    4. **test_client_handles_invalid_json** — Mock recv to return `b'not-json\n'`. Verify error_occurred.emit is called with a message containing "JSON decode error". Verify data_received.emit is NOT called.

    5. **test_client_detects_server_disconnect** — Mock recv to return `b''` (empty = server closed). Verify the method exits (connected set to False).

    **Important:** Patch `PySide6.QtCore.QThread.__init__` if needed to avoid Qt initialization. Or use `unittest.mock.patch` on the Signal class. The key is testing the parsing logic, not the Qt threading.
  </action>
