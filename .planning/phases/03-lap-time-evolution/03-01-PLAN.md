  <action>
Create a new PitWallWindow subclass `LapTimeEvolutionWindow` that:

1. **Data accumulation in on_telemetry_data():**
   - Track each driver's current lap number and the timestamp when they started it
   - When a driver's lap number increases, compute lap_time = current_t - lap_start_t
   - Store completed laps as a list per driver: `{lap_num, lap_time_s, compound_int}`
   - Track pit stops: when a driver's tyre compound changes between consecutive laps, mark that lap as a pit stop
   - Track safety car periods: accumulate (start_lap, end_lap) from track_status changes (status "4" = SC, "6"/"7" = VSC)
   - Skip lap 1 (formation/out lap) — only record from lap 2 onwards
   - Filter out outlier laps > 150s (pit laps, red flags)

2. **UI setup_ui():**
   - Top row: QListWidget for multi-driver selection (checkable items), sized ~150px wide
   - Left: driver list. Right: Matplotlib figure
   - Use QSplitter for driver list | chart layout
   - Matplotlib figure with single axes, dark background (#282828)
   - X-axis: Lap Number, Y-axis: Lap Time (seconds)
   - Title: "Lap Time Evolution"

3. **Compound colors for markers:**
   - SOFT (0): #FF3333 (red)
   - MEDIUM (1): #FFC300 (yellow)
   - HARD (2): #FFFFFF (white)
   - INTERMEDIATE (3): #43B02A (green)
   - WET (4): #0072CE (blue)

4. **Chart rendering (_redraw method):**
   - For each selected driver: scatter plot of lap_time vs lap_number
   - Each point colored by compound
   - Connect points with thin lines in driver team color (use a default palette of 10 distinct colors, cycle)
   - Mark pit stop laps with a vertical dashed gray line
   - Shade SC periods with yellow alpha band (axvspan)
   - Shade VSC periods with orange alpha band
   - Y-axis auto-scales to data range with 2s padding
   - X-axis from 1 to max lap

5. **Refresh on driver selection change** — connect QListWidget.itemChanged signal to _redraw

Window title: "F1 Race Replay - Lap Time Evolution"
Window size: setGeometry(100, 100, 1200, 700)
